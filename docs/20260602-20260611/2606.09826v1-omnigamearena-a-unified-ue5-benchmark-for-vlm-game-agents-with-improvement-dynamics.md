## 摘要
视觉-语言模型（VLM）代理越来越多地部署在交互式游戏环境中。然而，针对VLM代理的游戏基准通常只报告每个（代理，游戏）对的单次首次尝试得分，专注于单代理单人游戏，并且缺乏统一的协议来平等地评估异构代理类别（商业VLM、开放权重VLM和专门的游戏策略）。我们通过OmniGameArena和Improvement Dynamics Curve（IDC）来解决这些差距。OmniGameArena是一个实时基准，包含十二个新构建的Unreal Engine 5游戏，涵盖单人（7个）、玩家对玩家（3个）和合作（2个）模式，具有统一的动作接口。IDC是一个代理反思框架，其中使用工具的反射LLM在多个轮次中自主优化有界技能提示。除了冷启动排行榜分数外，IDC还为每个（代理，游戏）对暴露了两个额外的可观察量：分数在反思轮次中的演变，以及所学技能在保留任务变体上的行为。我们报告了十二个VLM代理在冷启动排行榜上以及四个顶级代理在IDC下的这些可观察量。

## Abstract
Vision-language model (VLM) agents are increasingly deployed in interactive game environments. Yet game benchmarks for VLM agents typically report a single first-attempt score per (agent, game) pair, focus on single-agent Solo play, and lack unified protocols for evaluating heterogeneous agent classes (commercial VLMs, open-weight VLMs, and specialized game policies) on the same footing. We address these gaps with OmniGameArena, a real-time benchmark of twelve newly built Unreal Engine 5 games spanning Solo (7), PvP (3), and Coop (2) with unified action interfaces, and the Improvement Dynamics Curve (IDC), an agentic-reflection harness in which a tool-using reflector LLM autonomously refines a bounded skill prompt across multiple rounds. Beyond cold-start leaderboard scores, IDC exposes two additional observables for each (agent, game) pair: how the score evolves across reflection rounds, and how the learned skill behaves on held-out task variants. We report these observables for twelve VLM agents on the cold-start leaderboard and four top agents under IDC.

### 2. 论文提出的方法论

    *   **统一接口**：所有游戏对外暴露统一的动作接口（键盘-鼠标、游戏手柄适配器），使商用 API 模型、开放权重模型、专用游戏策略都通过完全相同的指令通道与环境交互。
    *   **防泄漏设计**：采用“网页曝光审计”检查名称、规则是否已出现在公开语料中，并设计全新的关卡组合、脚本逻辑与成功标准，使**游戏识别率为 0%**，最大程度降低预训练数据污染风险。
    *   **连续评估指标**：游戏进度统一归一化到 \([0,1]\) 连续尺度（如位移比例、存活时间占比、伤害完成比等）。

*   **IDC 反思框架**
    *   **多轮自我改进**：每一轮让智能体在固定的技能提示下运行 \(K\) 个 episode，之后由一个**具备工具调用能力的反射 LLM** 检查轨迹、诊断失败、最终提炼出新的技能提示 \(m_{r+1}\)。
    *   **四阶段反射模块**：
        1.  **Explore**：使用只读工具（列出目录、读文本/图片、搜索）自主选择要检查的轨迹片段。
        2.  **Diagnose**：显式地提交失败原因列表与分析。
        3.  **Validate**：生成改进的技能提示，并调用独立评判 LLM 检查是否合规（如是否包含地图记忆内容），最多迭代 5 次。
        4.  **Distill**：将接受的技能存入经验笔记本，形成持久的记忆。
    *   **最优技能回滚**：若某轮得分低于历史最优值的一半，下一轮自动从最优技能恢复，防止技能灾难性漂移。
    *   **输出的可观察量**：除了冷启动分数 \(S_0\)，IDC 输出完整的**分数演化曲线** \([S_0, S_1, \dots, S_R]\)，以及将最优技能用于**保留任务变体**时的迁移分数。

### 3. 实验设计

*   **智能体类别与对比方法**
    *   **商业 VLM（API）**：Claude Opus 4.7/4.6、Claude Sonnet 4.6、GPT-5.5、GPT-5.4、Gemini 3.1 Pro/Flash-Lite Preview、Kimi K2.5。
    *   **开源 VLM**：Qwen3.5‑397B‑A17B、Qwen3.5‑122B‑A10B（本地部署）。
    *   **专用游戏策略**：NitroGen（手柄动作生成）、Open‑P2P（键鼠历史到动作）。
*   **评测基准与游戏场景**：全部 12 款 OmniGameArena 游戏，覆盖 7 个单人任务、3 个 PvP 对抗和 2 个合作任务。
*   **实验阶段**
    *   **冷启动排行榜**：所有 12 个智能体在每款游戏上以无技能提示的姿态各运行 **5 个 episode**，PvP 额外覆盖所有两两组合配对。采用两种时钟模式：**Paused Decision Quality (PDQ)**（冻结环境，只评估决策质量）与 **Latency‑Controlled Real‑Time (LCRT)**（注入模型端推理延迟，评估受延迟影响的表现）。
    *   **IDC 改进动态**：挑选冷启动表现最好的 **4 个智能体**（Claude Opus 4.6/4.7、GPT‑5.5、Gemini 3.1 Pro），在 LastStand（单人）和 SharedFloor（合作）两个游戏上进行 **R=10 轮**反思（每轮 \(K=5\) 个 episode）。测得完整 IDC 曲线以及最佳技能在 **3 种未知变体**上的迁移表现。
    *   **定性对比**：展示有无 IDC 反思的行为轨迹变化。

*   **实验覆盖面**：

*   **公平性**：所有智能体共享统一的 UE5 实时环境和动作接口，使用相同的 episode 数量和评估协议；PvP 采用对称的 LCRT 规则避免网络延迟的混淆。

### 6. 论文的主要结论与发现

*   **没有单一的统治者**：冷启动排行榜上，不同游戏的最强模型频繁轮换（如 GPT‑5.5 在四个单人游戏领先，Claude Opus 4.6 在 CueChase 领先巨大，Gemini 3.1 Pro 在 MonsterShoot 领先）。
*   **新版本并非总是更好**：Claude Opus 4.6 在多个单人任务上反超更新的 4.7，能力排名高度依赖于具体任务。
*   **开源与专用策略严重落后**：开源 VLM 和专用游戏策略在多款游戏上得分为 0，与商业模型之间存在显著鸿沟。
*   **反思确实带来提升但非单调**：四款模型在 IDC 下均获得超过冷启动基线的提升（单人存活分数提升幅度达 130%–437%），但峰值性能往往在**中盘**出现，后期有下降趋势，说明最优技能回滚机制的必要性。
*   **原始任务提升 ≠ 泛化能力**：LastStand 的原任务增益与变体迁移增益之间存在明显的分离。例如，GPT‑5.5 原任务提升最小，但却实现了全部三种变体的正向迁移；其他模型原任务改进大，却因为学到“静止不动”的脆弱策略而导致在变体上剧烈下降。SharedFloor 则因技巧聚焦于通用的协调规则（分工、避让等），表现出一致的正向迁移。

### 7. 优点

*   **全新构建的免污染环境**：从 0 开始设计 12 款 UE5 游戏，并证实了 0% 的识别率，极大增强了评测的客观性。
*   **多模式全覆盖**：首次将单人、PvP、合作三种交互模式集成到统一的实时基准中，并能测出如 PvP 中的“非传递性”（Kimi 击败了强单人对手 Claude）。
*   **IDC 框架的独创性**：不仅看冷启动分数，更测量改进曲线和迁移能力，且反射过程由 LLM 自主决定探查什么，避免了刻板的固定脚本。
*   **可比的异构接口**：通过适配器将VLMs 和专用策略都纳入同一键鼠/手柄空间，使得公平比较成为可能。
*   **两维度的延时控制**：PDQ 和 LCRT 将决策质量和实际部署的实时性分离开来，揭示出延迟对生存任务甚至可能有正向作用。

### 8. 不足与局限

*   **IDC 覆盖范围有限**：IDC 实验仅在 2 个游戏（单人 LastStand, 合作 SharedFloor）和 4 个智能体上进行，未能推广到更多游戏和 PvP 场景。
*   **单技能制**：反射器每次只维护一个有限的技能提示并覆盖，尚未探索像 Voyager 那样的技能库积累和组合。
*   **模型同质角色**：玩家和反射者均由相同底层模型担任，未测试“弱玩家+强反射者”等不对称架构下的改进行为差异。
*   **物理动作空间限制**：动作输出为离散的键鼠/手柄指令块，可能限制了需要连续精细控制的游戏的发挥。
*   **未探讨更多开源模型**：Qwen 之外的更多开放权重 VLM 没有被纳入冷启动对比。
