## 摘要
机器人足球是多智能体强化学习的一个具有挑战性的测试平台，因为它结合了部分可观测性、合作与对抗交互、稀疏奖励和长时域战术行为。RoboCup 2D足球仿真（RCSS2D）提供了一个成熟的机器人足球平台，然而其面向竞赛的服务器-客户端架构难以直接与现代基于Python的MARL工作流一起使用。我们提出R2D-RL，一个强化学习环境，它通过共享内存通信和周期级同步将RCSS2D和基于HELIOS的玩家客户端连接到Python MARL接口。R2D-RL支持全场地和基于场景的训练，具有可配置的对手、基本离散和混合参数化动作空间、动作掩码、基于期望控球价值（EPV）的奖励塑形以及并行执行。我们提供了前场进攻场景和一个11对11全场地基准，并给出了基线结果。

## Abstract
Robot soccer is a challenging testbed for multi-agent reinforcement learning because it combines partial observability, cooperative and adversarial interaction, sparse rewards, and long-horizon tactical behavior. RoboCup 2D Soccer Simulation (RCSS2D) provides a mature robot-soccer platform, but its competition-oriented server-client architecture is difficult to use directly with modern Python-based MARL workflows. We introduce R2D-RL, a reinforcement learning environment that connects RCSS2D and HELIOS-based player clients to a Python MARL interface through shared-memory communication and cycle-level synchronization. R2D-RL supports full-field and scenario-based training with configurable opponents, Base discrete and Hybrid parameterized action spaces, action masks, expected possession value (EPV)-based reward shaping, and parallel execution. We provide front-goal scenarios and an 11-vs-11 full-field benchmark, together with baseline results.

- **核心思想**：通过共享内存通信和周期级同步，将原始的RCSS2D服务器、修改后的HELIOS玩家客户端、教练、训练器进程与Python端的MARL接口桥接，构建一个“步骤同步”的学习环境。

### 6. 论文的主要结论与发现
- **环境可行性**：R2D-RL成功地将RCSS2D改造为了一个支持并行采样、步骤同步、可复现的全场足球MARL环境，并提供了一个从简单场景到全场比赛的递进式基准测试套件。

  - **MAPPO** 在多数场景中表现最稳定。

- **全场基线**：现有基线算法在与固定规则对手的11对11比赛中仍为负进球差，说明该基准仍具有极大的挑战性，为未来研究提供了基准。

### 7. 优点
- **环境设计精巧**：通过共享内存和周期同步，无需重写繁重的RCSS2D逻辑，便将其完整地包装成了现代MARL环境，设计思路巧妙。
- **基准设计系统性**：从单攻击手的“单元测试”到多攻击手的“集成测试”，再到全场的“系统测试”，为算法研究提供了清晰的递进式评估路径。
- **动作掩码设计**：将HELIOS内部的知识规则化为动态的动作掩码，是连接专家知识与学习算法的良好实践，尤其在减少无效探索上效果显著。
- **环境接口完整**：提供了全局状态、局部观测、动作掩码、奖励塑形、场景重置等完整的MARL环境语义。

### 8. 不足与局限
- **对手单一**：当前基线实验仅与固定的HELIOS Fallback规则对手对战，战术多样性不足，无法评估泛化能力。未来需引入自博弈或多样化的对手库。

- **决策范围受限**：学习仅限于`play_on`阶段，无法处理开球、任意球等定位球场景，而这些是足球策略的重要组成部分。
- **计算资源未透明化**：训练过程的GPU型号、数量、并行策略及具体耗时均未说明，难以估算复现所需的真实算力成本。
