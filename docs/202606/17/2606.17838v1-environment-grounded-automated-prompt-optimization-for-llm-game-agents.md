## 摘要
在交互环境中，大语言模型智能体对提示高度敏感，但提示工程仍是一个手动、特定于任务的过程。我们提出了一种自动提示优化框架，将观测到动作的流水线分解为目标条件描述符智能体和动作选择智能体，并通过由环境回报引导的LLM驱动的进化循环迭代优化每个模块的提示。我们设计了一个行为分析器，将回合结果归因于特定的提示组件，以及一个变异器，提出有针对性的提示修订，然后通过环境推演进行验证。我们在BALROG基准的全部五个BabyAI任务上进行评估，将我们的流水线与BALROG的RobustCoTAgent在普通和引导提示初始化下进行比较。优化在任务和条件中持续提高性能，无需更新模型权重。在PutNext这一多步协调任务中，RobustCoTAgent的成功率为0%，而我们的框架使用相同的底层LLM，通过优化提示达到了高达72.5%的成功率。这些结果表明，多智能体框架与自动提示优化相结合，无需微调或大量人工监督即可增强大语言模型。

## Abstract
LLM agents in interactive environments are highly sensitive to their prompts, yet prompt engineering remains a manual, task-specific process. We introduce an automated prompt optimization framework for LLM agents that decomposes the observation-to-action pipeline into a goal-conditioned descriptor agent and an action selection agent, and iteratively refines each module's prompt through an LLM-driven evolutionary loop guided by environment returns. We propose a behavior analyzer to attribute episode outcomes to specific prompt components, and a mutator to propose targeted revisions to the prompt, before validating them through environment rollouts. We evaluate on all five BabyAI tasks in the BALROG benchmark, comparing our pipeline against BALROG's RobustCoTAgent under both plain and guided prompt initializations. Optimization improves performance consistently across tasks and conditions, without requiring updates to the model weights. On PutNext, a multi-step coordination task where the RobustCoTAgent achieves 0% success, our framework reaches up to 72.5% success rate using the same underlying LLM with optimized prompts. These results suggest that a multi-agent framework, combined with automatic prompt optimization, enhances LLMs without the need for fine-tuning or extensive human supervision.

### 2. 论文提出的方法论

- **核心思想**：将单一的整体LLM智能体**分解**为两个子智能体以分摊负载：
    - **描述符智能体（Descriptor Agent, \(A_{\text{des}}\)）**: 专注于当前状态，提取与任务目标相关的信息，不负责规划。
    - **动作选择智能体（Action Selection Agent, \(A_{\text{sel}}\)）**: 基于描述符生成的摘要来选择下一步动作。
- **优化框架：RAPOA（Reward-driven Automatic Prompt Optimisation for Agentic systems）**
    - **形式化**：将问题建模为部分可观测马尔可夫决策过程（POMDP），目标是寻找一组最优提示 \(P^*\) 以最大化期望折扣回报。
    - **优化流程（进化循环）**:
        1.  **收集轨迹**: 在当前提示下，让智能体与环境交互并收集回合轨迹（包含状态、动作、奖励）。
        2.  **行为分析（Behavior Analyzer, BA）**: 一个LLM模块，分析失败/成功的轨迹，将结果（特别是失败原因）归因于特定的子智能体（描述符或动作选择器），生成一个按严重性排序的修改建议列表。
        3.  **提示变异（Mutator）**: 另一个LLM模块，根据BA的建议对特定子智能体的提示进行有针对性的修改（增加、修改或删除指令）。
        4.  **两阶段评估与验证（Acceptance Criteria）**:
            - **阶段一**: 在优化种子（\(S_{\text{opt}}\)）上评估变异后的智能体，要求其平均表现超过当前最佳提示一个阈值 \(\delta\)。
            - **阶段二**: 若通过第一阶段，则在独立的选择种子（\(S_{\text{select}}\)）上进行评估，以防止过拟合于特定的优化轨迹。
        5.  **迭代**: 只有通过两阶段评估的变异才会被接受，然后开始新一轮的轨迹收集和优化。

### 3. 实验设计

- **环境与数据集**: 使用 **BALROG 基准**中的全部五个 **BabyAI** 任务。

- **对比方法（基线）**:
    - **BALROG 提供的 RobustCoTAgent**:

    - **初始化**: 普通提示和引导式提示。

5.  **分析表明**，行为分析器能正确地将大部分失败归因于动作选择器，且模块特定的优化是有效的。

### 7. 优点

- **无需权重更新**: 该方法通过黑盒方式优化提示，适用于无法微调的闭源或超大模型。
- **自动化程度高**: 消除了提示工程中的人工干预，框架能根据环境反馈自动发现并修正问题。
- **架构灵活**: 提出的双模块（描述符+动作选择）和优化循环可以作为通用组件，与其它智能体结构（如规划模块）结合。
- **实验详尽扎实**: 从任务、基线、消融、超参数、到跨任务泛化性，实验设计非常全面，分析深入。
- **引入环境交互作为优化信号**: 相比于在静态数据集上优化提示，该方法利用了交互环境的回报，更难被过拟合，信号更真实。

### 8. 不足与局限

- **测试领域单一**: 所有实验均在一个环境（BabyAI网格世界）和一种LLM（GPT-OSS20B）上完成，其结论在其他更复杂的、开放域的环境或不同模型上的泛化能力有待验证。
- **计算成本较高**: 基于进化循环的方法需要大量与环境交互的样本（在线评估），对于更复杂的任务或更长轨迹，环境交互成本（时间和算力）可能过高。
- **优化架构简单**:
    - 仅使用单一个体进化策略，未探索如种群优化、交叉操作等更高级的算法，可能会陷入局部最优。
    - 行为分析器和变异器的元提示是手工设计的，这可能引入设计者的偏见。
- **依赖手工初始提示**: 虽然目标是从优化中获益，但实验表明起始提示的质量（如引导式优于普通提示）对最终性能有显著影响。方法本身没有解决如何从极简提示自动生成高质量引导式提示的问题。
- **奖励稀疏性挑战**: 在奖励极度稀疏的任务（如PutNext）中，优化过程对超参数（如阈值）和评估噪声非常敏感。
