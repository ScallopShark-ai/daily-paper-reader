## 摘要
本文提出一种将时序行为树解释为用于强化学习的奖励Petri网的方法。为复杂、长期运行的机器人任务设计奖励函数十分困难，特别是当任务具有层次结构和时序约束时。时序行为树扩展了机器人应用中使用的传统行为树，将时序属性引入其叶节点。这使得时序行为树不仅能表示由顺序、回退、并行等行为树算子定义的行为任务结构，还能表达任务的时序约束。本文中，约束在叶节点中使用线性时序逻辑指定。为了利用时序行为树来指导强化学习的奖励，我们提供了从时序行为树到Petri网的转换，并展示了如何根据时序行为树的结构自动分配奖励，从而得到奖励Petri网。在一系列难度递增的环境中，我们展示了基于时序行为树的奖励如何在普通强化学习方法失败的情况下实现学习，提高样本效率，并对学习进程提供灵活、直观的控制。我们通过使用不同的奖励分配方案和时序行为树结构，展示了其对学习的影响。

## Abstract
This paper introduces an interpretation of Temporal Behavior Trees (TBTs) as Reward-Petri-Nets (RPNs) for reinforcement learning (RL). Designing reward functions for complex, long-horizon robotic tasks is notoriously difficult, especially when tasks have hierarchical structure and temporal constraints. TBTs extend conventional behavior trees (BTs) used in robotic applications by incorporating temporal properties into their leaf nodes. This allows TBTs to represents not only the behavioral task structure defined by BT operators such as Sequence, Fallback, and Parallel, but also the task's temporal constraints. In this work, the constraints are specified in the leaf nodes using Linear Temporal Logic. In order to inform RL rewards using TBTs, we provide a translation from TBT into a Petri Net (PN) and show how rewards can be automatically assigned based on the TBT's structure, resulting in a RPN. In a series of increasingly challenging environments, we demonstrate how TBT-based rewards enable learning where vanilla RL fails, improve sample efficiency, and offer flexible, intuitive control over the learning progress. We showcase the learning impact by using different reward distribution schemes and TBT structures.

## 1. 论文的核心问题与整体含义
- **研究动机**：复杂、长时域机器人任务中，设计奖励函数面临巨大困难，尤其是当任务包含层次化结构和时序约束时。标准稀疏奖励下，强化学习（RL）往往无法有效探索或分配信用。
- **背景**：虽有人利用形式化规范（如自动机）自动生成奖励，但这类方法减少了用户的控制权，难以体现子任务重要性的差异。时序行为树（TBT）将传统行为树与叶节点的线性时序逻辑结合，既能保留行为树的结构化和模块化优势，又能表达时序约束。
- **核心目标**：将 TBT 解释为奖励 Petri 网（RPN），利用其结构自动生成密集奖励，从而在用户可控的前提下，提升强化学习的样本效率与任务解决能力。

## 2. 方法论
- **TBT 语法**：树结构由叶节点 `Leaf(φ)`（φ 是 LTL 公式）和组合算子 `Fback`（回退）、`Par_M`（并行，至少 M 棵子树成功）、`Seq`（顺序）构成。
- **Petri 网构造**：递归地将 TBT 翻译为 Petri 网。叶节点对应的库所通过 guard 控制：监视器使用 LTL₃ 语义的自动机，当返回真时触发 `t_φ`，假时触发 `t_¬φ`。控制流通过同步库所（如 `psync`）和控制库所（`cT`）实现，保证满足 TBT 语义。
- **奖励 Petri 网（RPN）**：在 Petri 网基础上加入：
  - **Guard 与动作**：为每个叶节点转换绑定自动机的启动/停止动作，实现可“回溯”的失败重试（但可能引入不精确性，见定理 1）。
  - **奖励分配函数**：定义节点类型的奖励分配函数 `fn_Leaf, fn_Fback, fn_Par, fn_Seq`，通过递归函数 `d_T(r)` 将根节点的总奖励 `r` 向下传播到具体转换上，支持用户自定义子任务权重。
- **MDP 嵌入**：MDP 状态扩展为环境状态与 RPN 标记的联合状态，每步根据监视器更新标记并累加发射的奖励。

## 3. 实验设计
- **环境与基准**：使用 MiniGrid 框架，包含 5 个难度递增的网格世界任务：Unlock（开锁）、UnlockPickup（开锁并拾取）、BlockedUnlockPickup（搬障碍后开锁拾取）、ObstructedMaze‑2Dlh（多门迷宫）、ObstructedMaze‑1Q（带障碍的多门迷宫）。最后两个属于硬探索问题，此前没有成功使用 PPO 的报道。
- **对比方法**：
  - **Vanilla‑baseline**：原始 MiniGrid 稀疏奖励。
  - **TBT‑baseline**：将 TBT 状态加入观察，仅在根节点满足时给予单位奖励（稀疏的 RPN 奖励）。
  - **TBT‑Uniform**：单位奖励在 TBT 所有节点上均匀分配（密集奖励）。
  - **TBT‑MonInc**：奖励沿 RPN 单调递增地分配给叶节点，让后期子目标获得更高奖励（密集奖励）。
  - 所有 TBT 方案均包含失败惩罚 `R(t_¬T) = -0.01 × R(t_T)`，并加入恒定步惩罚 `r_step = -0.9/n_max_steps` 以激励行动。
- **算法与评估**：采用 PPO（Stable Baselines3），每组实验运行 5 个种子，以平均 MiniGrid 原始回报作为评价指标，训练曲线展示均值与极值区间。

## 6. 主要结论与发现
- **解决困难任务**：TBT 导出的 RPN 奖励可使 vanilla RL 完全失败的任务被成功学习。
- **样本效率**：与 Vanilla‑baseline 相比，TBT‑based 方法显著提高了样本效率；即使在稀疏 TBT‑baseline 下也有一定提升，但方差较大。
- **奖励控制**：用户通过选择奖励分配方案（如 MonInc 对 ObstructedMaze 学习速度有轻微优势）和调整 TBT 结构，可灵活、直观地控制学习进程。
- **可扩展性**：方法能随环境复杂度增长而平稳扩展，在硬探索问题上表现稳定，训练所需 episode 数控制在 3k~13k 范围内（Unlock 仅需约 500 episodes）。

## 7. 优点
- **形式化与模块化**：TBT 表达力强于常用 LTL/STL，且保持可读性和可组合性，有利于人机交互设计。
- **自然映射**：Petri 网能自然表示并行、顺序和回退，支持可回溯的失败处理，避免状态空间爆炸。
- **灵活可控**：奖励分配函数允许用户定义子任务的重要度，兼顾自动化生成与人工直觉。
- **实验系统**：在阶梯式难度环境上系统验证，并与稀疏、均匀、递增三种密度方案对比，论证清晰。
