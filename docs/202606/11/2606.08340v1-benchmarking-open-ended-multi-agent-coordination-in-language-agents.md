## 摘要
随着语言模型越来越多地被部署为自主智能体，它们必须在开放式交互任务中与他人进行长期协调。然而，现有的评估很少同时测试这些需求，而是侧重于单智能体任务、短交互或高度结构化的多智能体设置。我们引入了$alem$，一个基于JAX的开放式多智能体协调基准，构建在类似Craftax的动力学之上。Alem将程序生成的协调任务、软专业化、通信和可控的协调难度嵌入到一个具有探索、制作、交易和战斗的长期生存世界中。我们评估了13个现代LLM在零样本条件下在同类团队中的表现，并以训练有素的MARL智能体作为参考点。当前的LLM智能体离解决alem还很远，平均标准化回报仅为约6%，但它们的失败并不均匀。在最难的协调设置下，零样本Gemini-3.1-Pro-High接近训练了十亿步的MARL智能体，而GPT-5.4-High在基础任务上获得了高奖励，但协调奖励却低得多。这种对比表明，个体任务能力并不意味着协调能力。消融实验表明，通信是协调的最大贡献者，而记忆和推理在用于维护多步计划时会有所帮助。总的来说，我们的结果指出协调是前沿LLM智能体的一个明显瓶颈，与单智能体能力分开。Alem使这一瓶颈可度量，并为开发能够通信、分配角色和执行共享计划的智能体提供了一个受控测试平台。代码可在https://github.com/alem-world/alem-env获取。

## Abstract
As language models are increasingly deployed as autonomous agents, they must coordinate with others over long horizons in open-ended interactive tasks. Yet existing evaluations rarely test these demands together, instead emphasising single-agent tasks, short interactions, or highly structured multi-agent settings. We introduce $alem$, a JAX-based benchmark for open-ended multi-agent coordination built on Craftax-like dynamics. Alem embeds procedurally generated coordination tasks, soft specialisation, communication, and controllable coordination difficulty into a long-horizon survival world with exploration, crafting, trading, and combat. We evaluate $13$ modern LLMs zero-shot within homogeneous teams, with trained MARL agents as reference points. Current LLM agents remain far from solving alem, averaging only ~6% normalised return, but their failures are not uniform. On the hardest coordination setting, zero-shot Gemini-3.1-Pro-High approaches MARL agents trained for one billion steps, while GPT-5.4-High achieves strong base-task reward but much lower coordination reward. This contrast shows that individual task competence does not imply coordination competence. Ablations show that communication is the largest contributor to coordination, while memory and reasoning help when used to maintain multi-step plans. Overall, our results identify coordination as a distinct bottleneck for frontier LLM agents, separate from single-agent capabilities. Alem makes this bottleneck measurable and provides a controlled testbed for developing agents that communicate, allocate roles, and execute shared plans. Code is available at https://github.com/alem-world/alem-env.

- **软专业化与难度控制**：
  - 任何智能体均可尝试受角色限制的动作，但非专长者成功概率 \(\eta_\text{non} = 1 - \alpha\)。

## 5. 实验数量与充分性
- **LLM 评估**：13 个模型 × 3 个难度，开放模型 20 种子/闭源 10 种子，共数百次完整回合评估，覆盖面广。
- **MARL 基线**：4 种算法 × 2 个训练步数（1亿/10亿） × 3 难度，额外将最强算法扩展到 30 亿步，训练充分。
- **消融与诊断**：针对 Gemini 和 Gemma 进行了通信、记忆、推理三项消融；还分析了死亡原因、成就层级、专长使用、协调事件频率等，多角度佐证结论。
- **公平性**：所有 LLM 使用相同的文本接口和提示范式，MARL 在同一环境超参数调优，对比基础统一。

## 6. 主要结论与发现
- **协调是独立瓶颈**：LLM 平均总奖励仅 ~6%，但基础任务能力不保证协调能力。GPT-5.4 基础分高但协调分低；Gemini-3.1 在 Hard 难度协调分（17.5%）接近训练十亿步的 MARL 智能体（17.6%）。
- **通信贡献最大**：移除通信导致协调奖励急剧下降（如 Gemini 从 17.5% 降至 5.3%），而基础奖励变化较小。
- **记忆与推理的作用依赖用法的质**：Gemini 用记忆编写前向计划（多行动序列）时效果较好，Gemma 多用记忆复述当前状态时效果有限。降低推理能力会削弱计划式行为，且记忆无法完全补偿。
- **异质团队未达最强成员水平**：混合团队表现接近各同质团队平均值，说明简单加入强模型并不能自动提升团队协调性能。
- **不同协调类型难度分化**：同步硬协调最难，交接任务相对容易，建造类稀疏协调对 LLM 有微弱优势但 MARL 难以发掘。

## 7. 优点
- **协调难度的可控性**：用单一标量 \(\alpha\) 调节协调强度且保持世界布局不变，可干净地分离执行难度与任务变化。
- **全谱系协调集成**：在同一环境内实现长期依赖、同步、交接等多类协调，避免环境碎片化。
- **过程化生成**：每次世界和协调结构都重新采样，减少记忆效应和污染风险，促进泛化性评估。
- **多模态接口**：同时提供像素、符号、文本界面，方便强化学习与语言智能体的统一对比。
- **快速且可复现**：基于 JAX 的并行化设计大幅降低大规模实验成本，单 GPU 即可完成十亿步训练。
- **深入诊断**：信息论验证、成就层级、事件统计、消息风格分析等使失败原因可解释。

## 8. 不足与局限
- **LLM 覆盖范围**：未评估视觉语言模型（VLM）和 Claude 系列模型，结论主要针对纯文本 LLM。
- **种子数量限制**：受 API 成本约束，闭源模型每难度仅 10 个种子，统计效力可能稍弱。
- **交流通道有限**：当前仅支持离散广播消息，更丰富的即时语音或连续通信工具未测试。
- **代理架构简单**：采用固定 prompts 和零样本“思考-行动”范式，未探索更复杂的结构化记忆、反思或跨回合学习。
- **异质团队实验初探**：仅测试了两种混合组合，且未能深入诊断混合团队中的协调失败机理。
- **应用限制**：Alem 仍是一款游戏基准，与现实多智能体协作任务（如机器人、办公自动化等）存在 domain gap。
- **未来工作**：可结合跨回合记忆、终身学习、更强的代理架构及更多样的模型家族以拓宽评估深度。
