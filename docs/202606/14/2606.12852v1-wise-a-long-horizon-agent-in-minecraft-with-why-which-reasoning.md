## 摘要
通过采用LLM增强的分层方法，在Minecraft等环境中开发通用具身智能体已取得快速进展。尽管前景广阔，但低级控制器常因反复执行失败而成为性能瓶颈。我们认为，关键限制不仅在于缺乏情节记忆，还在于“什么-哪里-何时”记忆与“哪个-为何”推理的脱节。为解决此问题，我们提出WISE（Which-Why Informed Semantic Explorer，即“哪个-为何”信息语义探索者），这是一个长期任务智能体框架，其增强的低级控制器配备了一个因果事件图，通过将观察与任务相关性联系起来的显式因果结构来增强情节记忆。与之前依赖于特征相似性进行检索的工作（如MrSteve）不同，WISE能够在视角变化下实现稳健回忆，并通过因果推理支持机会主义任务重排序。基于此记忆，我们提出了一种机会主义任务调度器，在检测到因果相关机会时动态重新排列子任务的优先级。我们进一步为WISE配备了一种多尺度渐进探索策略，为下游推理提供空间上全面的观察。实验表明，WISE在长期稀疏任务上大大提高了任务成功率和效率，特别是在需要自适应决策的环境中。

## Abstract
Rapid advances have been made in developing general-purpose embodied agent in environments like Minecraft through the adoption of LLM-augmented hierarchical approaches. Despite their promise, low-level controllers often become performance bottlenecks due to repeated execution failures. We argue that a key limitation is not only the lack of episodic memory, but also the decoupling of \textit{what-where-when} memory from \textit{which-why} reasoning. To address this, we propose \textbf{WISE} (Which-Why Informed Semantic Explorer), a long-horizon agent framework with an enhanced low-level controller equipped with a Causal Event Graph that augments episodic memory with explicit causal structure linking observations to task relevance. Unlike prior work such as MrSteve, which relies on feature similarity for retrieval, WISE enables robust recall under viewpoint changes and supports opportunistic task reordering through causal reasoning. Building on this memory, we propose an Opportunistic Task Scheduler that dynamically re-prioritizes subtasks when causally relevant opportunities are detected. We further equip WISE with a multi-scale progressive exploration strategy to provide spatially comprehensive observations for downstream reasoning. Experiments show that WISE largely improves task success and efficiency on long-horizon sparse tasks, particularly in settings requiring adaptive decision-making.

- **研究背景**：在 Minecraft 等开放世界环境中，构建能完成长期复杂任务的具身智能体仍是重大挑战。现有 LLM 增强的层次化方法将任务分解为高层规划与低层执行，但低层控制器常因缺乏有效记忆和灵活决策而成为瓶颈。
- **核心问题**：低层控制器不只是缺少情景记忆，而是记忆（“什么、哪里、何时”）与推理（“哪个、为何”）之间的脱节。以 MrSteve 为代表的方法仅依赖视觉特征相似性检索记忆，在视角变化下极易失败，且无法利用偶然遭遇的机会动态调整任务顺序，导致重复探索和低效。
- **整体含义**：论文提出 WISE 框架，将显式因果结构注入情景记忆，使智能体不仅能记住观察，还能理解“为什么这个观察重要”以及“它可能开启哪些未来任务”，从而在长期稀疏任务中实现更高效、更自适应的决策。

## 2. 论文提出的方法论

WISE 由三大核心模块构成，形成探索→记忆→规划的闭环。

- **因果事件图 (Causal Event Graph)**
  - **短期几何记忆**：沿用 MrSteve 的 Place Event Memory，用 MineCLIP 编码视觉帧，DP‑Means 聚类组织事件，按余弦相似度检索。
  - **VLM 驱动的图构建**：通过聚类和图像熵混合策略选取关键帧，由 GPT‑4o 异步提取语义实体、推断因果关系（如 `cow → CAN_OBTAIN → beef`）及空间共现关系，形成实体节点和因果边。
  - **两层检索**：同时利用视觉相似度和因果图匹配，加权评分（公式 1），使得即使视角变化也能根据因果链稳定检索相关记忆。

- **机会主义任务调度器 (Opportunistic Task Scheduler)**
  - 综合当前观察、待执行子任务集合以及因果事件图的记忆，为每个子任务计算优先级分数（公式 2）。
  - 优先级由紧急性、因果相关性（主导）和导航成本加权决定。当检测到因果相关实体出现时，动态重排任务队列（例如遇到牛则立即优先执行“获取牛肉”）。
  - 将低层控制器从僵硬序列执行器转变为可实时抓住机会的决策者。

- **多尺度渐进探索 (Multi‑Scale Progressive Exploration)**
  - **全局层**：四叉树分区，根据未探索深度和距离选择待访问区域，避免局部贪婪。
  - **区域层**：基于前沿点（frontier）检测，综合考虑覆盖增益、新异性和距离，引导扩展边界。
  - **局部层**：通过 Voronoi 分解识别并填补内部未探索空隙，确保覆盖完整性。
  - 三重层次保证探索既高效又全面。

- WISE 在探索覆盖率上大幅领先：真实小地图 10,000 步时覆盖率达 97%（MrSteve 67%），大地图 40,000 步达到 83%（MrSteve 69%）。

- 在 ABC‑Sparse 自适应任务上，WISE 成功率 77%（MrSteve 33%），完成步数缩短 42.5%，机会主义调度器成功捕获偶然出现的任务机会。

## 7. 优点

- **创新性强**：首次将“为何”推理（因果知识）引入低层控制器的情景记忆，弥补了“哪个‑为何”脱节问题。
- **闭环设计**：探索、记忆、调度三者相互增强，形成自我完善的系统，而非独立的模块堆砌。
- **实际效果显著**：在多种任务和地图尺寸下，相较最强基线 MrSteve，呈现出 30‑44 个百分点的成功率提升及显著的时间节约。
- **实验严谨**：通过丰富的对比和消融实验，清晰分离各组件贡献，排除了信息特权等混杂因素。
- **适应性突出**：机会主义调度使智能体能够在线利用意外遭遇，使执行更接近理想的最优路径。
- **可解释性**：因果图显式存储了“实体‑任务”关联，便于分析和调试。

## 8. 不足与局限

- **依赖外部大模型**：VLM（GPT‑4o）的调用成本（约 0.48 美元/回合）和异步延迟可能限制实时应用和大规模部署；论文未讨论轻量化替代方案。
- **真实环境覆盖率仍有差距**：在大地图真实环境中覆盖率仅 83%，远低于模拟环境的 98%，表明多尺度探索对动态干扰的鲁棒性有待提升。

- **任务范围**：实验中仅测试了相对少量的子任务组合（ABA、ABC），更复杂、更长链的任务（如工艺树多步合成）尚未验证。
- **与高层规划的结合**：论文重点在低层控制器，未深入探索因果图如何反馈优化高层 LLM 规划，闭环尚有延展空间。
- **可迁移性未验证**：仅在 Minecraft 环境中评估，对于其他具身环境的通用性尚待检验。
