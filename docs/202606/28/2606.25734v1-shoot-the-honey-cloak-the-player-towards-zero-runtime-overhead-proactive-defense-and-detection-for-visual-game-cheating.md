## 摘要
视觉自瞄作弊器已成为第一人称射击游戏中的严重作弊威胁，它们仅依赖渲染画面而非游戏内存运行，从而规避现有的反作弊防御。然而，现有防御无法提供端到端解决方案：事后行为检测器无法实时保护比赛完整性，且在面对模仿人类的作弊器时愈发脆弱；而主动运行时防御通常缺乏可问责性、带来显著开销或需要侵入式系统集成。我们提出AimTrap，首个针对视觉自瞄作弊器的端到端防御方案，通过两种对抗纹理机制将实时保护与赛后检测相结合。对抗伪装纹理（ACT）将真实玩家对作弊器隐藏，而对抗蜜罐纹理（AHT）诱使自瞄器锁定虚假目标，从而提供强有力的作弊证据。为使其实用，AimTrap集成了可微渲染与渲染期望，用于鲁棒的三维纹理合成、安全纹理管理，以及新颖的蜜罐交互轨迹分析流水线，以实现准确的作弊归因。在与最先进的视觉自瞄器的真实游戏评估中，ACT成功防御率达85.1%，AHT达96.9%。与先前基线相比，AimTrap取得了极低的误报率，同时仅带来可忽略的运行时开销。这些结果表明，AimTrap为视觉自瞄器提供了一种实用且有效的端到端防御。

## Abstract
Visual aimbots have emerged as a serious cheating threat in first-person shooter (FPS) games, as they evade existing anti-cheat defenses by operating only on rendered frames rather than game memory. However, existing defenses fail to provide an end-to-end solution: post-hoc behavior detectors cannot protect match integrity in real time and are increasingly fragile against human-mimicking aimbots, while proactive runtime defenses often lack accountability, incur substantial overhead, or require intrusive system integration.   We present AimTrap, the first end-to-end defense against visual aimbots that combines real-time protection with post-game detection using two adversarial texture mechanisms. Adversarial Camouflage Textures (ACT) hide real players from aimbots, while Adversarial Honeypot Textures (AHT) lure aimbots into locking onto fake targets, yielding strong evidence of cheating. To make this practical, AimTrap integrates differentiable rendering with Expectation over Renderings for robust 3D texture synthesis, secure texture management, and a novel honeypot-interaction trajectory analysis pipeline for accurate cheating attribution.   In real-game evaluation against a state-of-the-art visual aimbot, ACT achieves 85.1% defense success, AHT achieves 96.9%. Compared with prior baselines, AimTrap attains extremely low false-positive rates, while incurring negligible runtime overhead. These results show that AimTrap provides a practical and effective end-to-end defense against visual aimbots.

### 2. 方法论
#### 2.1 核心思想
- **两种互补的对抗纹理机制**：
  - **对抗伪装纹理（ACT）**：静态噪声贴于玩家模型，显著降低自瞄检测置信度（使玩家“隐身”），但对人类玩家视觉自然。
  - **对抗蜜罐纹理（AHT）**：静态诱饵嵌入游戏场景，对自瞄目标检测器呈现为高置信度“玩家”，诱骗其反复锁定假目标；对人类玩家则不可感知。
- **关键洞察**：将检测难题转化为**主动视觉挑战**——自瞄器对检测置信度敏感，而人类不会对蜜罐产生有意的瞄准行为；静态蜜罐通过相对运动在屏幕上表现为动态假目标。

#### 2.2 关键技术细节
- **三维鲁棒纹理合成（基于可微渲染 + 渲染期望 EoR）**
  - 使用端到端可微渲染管线，将纹理优化目标定义为在渲染条件分布（如视角、距离、光照）下的期望损失。
  - ACT 目标函数：最小化自瞄模型的检测置信度（逃避损失），并约束扰动在 ℓ∞ 球内以保持视觉自然性。
  - AHT 目标函数：最大化**唯一且高置信的诱骗框**，并加入几何一致性正则项（如边界框尺寸约束），避免多目标干扰。
  - 公式形式：\( \min_{\tau} \mathbb{E}_{\theta \sim D} \left[ \mathcal{L}(f(R(M,\tau;\theta)), y) \right] \)，通过可微渲染链式求导梯度至纹理像素。
- **安全纹理管理**：
  - 离线合成纹理池，每局随机抽样（类似 ASLR）。
  - **仅内存部署**：赛后立即清除，防止磁盘驻留和逆向。
  - **资产完整性校验**：防止篡改模型或 UV 映射。
- **蜜罐交互轨迹检测流水线**：
  - **轨迹提取**：基于几何门控（蜜罐邻近区域 HPZ、瞄准判定区 ADA）与时序聚类（1-D DBSCAN）从回放日志中提取 2D 瞄准轨迹。
  - **异常识别**：构建双向 LSTM 序列分类器，输入为时序 Δt、2D 坐标、命中蜜罐标志及蜜罐尺寸等，输出是否为作弊锁定。
  - **玩家级决策**：聚合一局内多个轨迹预测，超过阈值 T 即判为作弊（二项分布模型计算玩家级 TPR/FPR）。

- **防御有效性**：ACT 将平均检测置信度从 62.3% 降至 16.0%，逃避成功率 85.1%（SSIM 81.0%）；AHT 诱骗成功率 96.9%，唯一性率 96.9%，置信度 79.3%。

- **鲁棒性与转移性**：在多种视角、距离下性能稳定（AHT DSR 基本 >95%）；AHT 跨模型转移至 YOLOv5s 时 DSR 仍达 96.93%，但至 YOLOv5l 降至 58.79%。

### 7. 优点
- **端到端且零开销**：首次统一实时防御（无 per-frame 计算）和赛后可解释检测，部署友好，无需侵入渲染管线。
- **强信号设计**：蜜罐锁定是作弊特异性行为，解决模仿人类作弊器难以判别的问题，提供可解释证据。
- **三维鲁棒纹理**：利用可微渲染 + EoR 离线合成，保证纹理在 3D 游戏多变条件下的有效性，优于先前仅限 2D 平面的方法。
- **安全的纹理生命周期**：随机化、纯内存、完整性校验，提高攻击者逆向与适应难度。
- **实验全面且贴近实际**：包含实战作弊、真实玩家用户研究，并与顶尖检测器对比，验证了方法的实用性和低误报。

### 8. 不足与局限
- **地图与资产依赖**：检测需知晓蜜罐的 3D 坐标和尺寸用于投影变换，对官方加密地图难以实施（实验用自定义地图），实际部署需厂商配合。
- **转移性有限**：当自瞄模型与训练代理模型差异较大时（如 YOLOv5l），AHT 诱惑率下降明显（58.79%），可能需要重新训练纹理。
- **白盒假设**：合成时需自瞄检测器的梯度信息，若作弊者使用完全不可微的黑盒模型或频繁更新架构，纹理对抗性可能失效。
- **参与规模**：实战研究仅 20 人，虽涵盖不同经验，但样本量较小，玩家行为多样性可能不足；且仅在 CS2 单一地图测试，未在多地图、多游戏引擎验证。
- **潜在对抗适应**：作弊开发者可能通过针对性训练（如对抗训练）或蜜罐检测来规避，论文虽讨论了随机化和内存保护，但未进行深入的适应性攻击评估。
