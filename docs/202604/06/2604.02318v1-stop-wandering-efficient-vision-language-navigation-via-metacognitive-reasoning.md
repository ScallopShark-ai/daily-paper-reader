## 摘要
由基础模型驱动的免训练视觉语言导航 (VLN) 智能体能够遵循指令并探索 3D 环境。然而，现有方法依赖于贪婪的前沿选择和被动空间记忆，导致了局部振荡和冗余重复访问等低效行为。我们认为，这源于元认知能力的缺失：智能体无法监控其探索进度、诊断策略失败或进行相应的调整。为解决这一问题，我们提出了 MetaNav，这是一种集成了空间记忆、历史感知规划和反思性修正的元认知导航智能体。空间记忆构建了一个持久的 3D 语义地图；历史感知规划通过惩罚重复访问来提高效率；反思性修正能够检测停滞状态，并利用大语言模型 (LLM) 生成修正规则，以指导未来的前沿选择。在 GOAT-Bench、HM3D-OVON 和 A-EQA 上的实验表明，MetaNav 在实现最先进性能的同时，将视觉语言模型 (VLM) 的查询次数减少了 20.7%，证明了元认知推理显著提高了鲁棒性和效率。

## Abstract
Training-free Vision-Language Navigation (VLN) agents powered by foundation models can follow instructions and explore 3D environments. However, existing approaches rely on greedy frontier selection and passive spatial memory, leading to inefficient behaviors such as local oscillation and redundant revisiting. We argue that this stems from a lack of metacognitive capabilities: the agent cannot monitor its exploration progress, diagnose strategy failures, or adapt accordingly. To address this, we propose MetaNav, a metacognitive navigation agent integrating spatial memory, history-aware planning, and reflective correction. Spatial memory builds a persistent 3D semantic map. History-aware planning penalizes revisiting to improve efficiency. Reflective correction detects stagnation and uses an LLM to generate corrective rules that guide future frontier selection. Experiments on GOAT-Bench, HM3D-OVON, and A-EQA show that MetaNav achieves state-of-the-art performance while reducing VLM queries by 20.7%, demonstrating that metacognitive reasoning significantly improves robustness and efficiency.

*   **研究背景**：作者认为这些问题的根源在于智能体缺乏**元认知能力（Metacognitive capabilities）**。人类在探索时会监控进度、诊断失败原因并调整策略，而现有智能体大多采用“贪婪”策略，仅依赖即时感知或被动记忆，无法识别自己是否陷入了僵局。
*   **整体含义**：MetaNav 通过引入“感知-规划-反思”的闭环机制，赋予智能体自我监控和策略修正的能力，从而实现更高效、更稳健的 3D 环境导航。

### 2. 方法论：核心思想与技术细节
MetaNav 由三个核心模块组成，形成一个元认知导航循环：
*   **D1：空间记忆构建（Spatial Memory Construction）**
    *   利用 RGB-D 流，通过 TSDF（截断符号距离函数）融合技术构建持久的 3D 语义地图。
    *   将环境划分为已占据、空闲和未知空间，并提取“前沿（Frontiers）”（即已知空闲空间与未知空间的边界）作为潜在目标。
*   **D2：历史感知启发式规划（History-Aware Heuristic Planning）**
    *   **解耦决策与执行**：不逐帧调用 VLM，而是设定固定的重规划间隔（如每 3 步调用一次），降低延迟。
    *   **统一效用函数**：结合语义相关性（VLM 评分）、几何代价（距离）和**情节惩罚（Episodic Penalty）**。情节惩罚通过对已访问位置进行时间衰减的高斯建模，抑制智能体返回近期去过的地方。
*   **D3：反思与修正（Reflection and Correction）**
    *   **停滞检测**：监控“信息增益”（新探索的体积）。如果连续多步增益低于阈值，则触发停滞警报。
    *   **LLM 诊断**：触发后，调用 LLM 分析情节记忆（包括短时窗口和长时摘要），生成“避开（Avoid）”或“尝试（Try）”的修正规则。这些规则会反馈给规划器，干预后续的前沿选择。
