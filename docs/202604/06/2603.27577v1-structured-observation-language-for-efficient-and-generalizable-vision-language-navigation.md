## 摘要
视觉语言导航 (VLN) 要求具身智能体通过遵循自然语言指令在复杂环境中进行导航，这通常需要视觉和语言模态的紧密融合。现有的 VLN 方法通常将原始图像转换为视觉 token 或隐式特征，这需要大规模的视觉预训练，并且在环境变化（如光照、纹理）下泛化能力较差。为了解决这些问题，我们提出了 SOL-Nav（Structured Observation Language for Navigation），这是一个将第一视角视觉观测转化为紧凑结构化语言描述的新型框架，旨在实现高效且可泛化的导航。具体而言，我们将 RGB-D 图像划分为 N*N 网格，提取每个网格单元具有代表性的语义、颜色和深度信息以构建结构化文本，并将其与语言指令拼接，作为预训练语言模型 (PLM) 的纯语言输入。在标准 VLN 基准测试（R2R、RxR）和真实世界部署中的实验结果表明，SOL-Nav 显著降低了模型规模和对训练数据的依赖，充分利用了 PLM 的推理和表示能力，并在未知环境中实现了强大的泛化性能。

## Abstract
Vision-Language Navigation (VLN) requires an embodied agent to navigate complex environments by following natural language instructions, which typically demands tight fusion of visual and language modalities. Existing VLN methods often convert raw images into visual tokens or implicit features, requiring large-scale visual pre-training and suffering from poor generalization under environmental variations (e.g., lighting, texture). To address these issues, we propose SOL-Nav (Structured Observation Language for Navigation), a novel framework that translates egocentric visual observations into compact structured language descriptions for efficient and generalizable navigation. Specifically, we divide RGB-D images into a N*N grid, extract representative semantic, color, and depth information for each grid cell to form structured text, and concatenate this with the language instruction as pure language input to a pre-trained language model (PLM). Experimental results on standard VLN benchmarks (R2R, RxR) and real-world deployments demonstrate that SOL-Nav significantly reduces the model size and training data dependency, fully leverages the reasoning and representation capabilities of PLMs, and achieves strong generalization to unseen environments.

视觉语言导航（VLN）要求智能体根据自然语言指令在复杂环境中移动。现有方法主要面临三大挑战：
*   **训练成本高**：依赖大规模视觉预训练，从像素级特征开始训练视觉编码器，收敛慢。
*   **泛化性差**：模型容易过拟合特定的视觉外观（如光照、纹理、布局），在未见过的环境中表现不佳。
*   **模态鸿沟**：视觉 token 与语言模型之间存在对齐困难，难以充分发挥预训练语言模型（PLM）的逻辑推理能力。
**SOL-Nav 的核心动机**是通过将视觉观测直接转化为“结构化文本”，消除视觉编码器的负担，实现纯语言驱动的导航决策。

#### 2. 方法论：核心思想与技术细节
SOL-Nav 的核心思想是将 RGB-D 图像“翻译”成语言模型可理解的结构化描述。
*   **结构化观测构建**：
    *   **网格划分**：将图像划分为 $N \times N$ 的网格。
    *   **特征提取**：针对每个网格单元，提取三类信息：**平均深度**（空间距离）、**语义类别**（通过 SegFormer 等模型获取主导标签）、**代表性颜色**（HSV 转标准颜色名）。
    *   **文本转化**：格式化为 `[i,j]: depth=d, semantic=s, color=c`。
*   **多分辨率历史记忆**：
    *   **当前观测**：采用高分辨率网格（如 $6 \times 6$）。
    *   **短期历史**：中等分辨率（$4 \times 4$）。
    *   **长期历史**：低分辨率（$2 \times 2$），以压缩长序列长度。
*   **模型架构**：
    *   使用 **Qwen3-Embedding-0.6B** 作为骨干网络。
    *   **动作块预测（Action Chunking）**：模型一次性预测连续的 4 步动作（停止、左转、右转、前进），以提高决策的连贯性。
    *   **输入 Prompt**：由系统描述、结构化观测序列、任务指令组成纯文本输入。

#### 6. 主要结论与发现
*   **小模型胜大模型**：仅 0.6B 参数的 SOL-Nav 在 R2R-CE 上的表现优于许多 7B 规模的模型（如 UniNaVid），且无需额外的海量多模态数据预训练。
*   **极强的 OS 表现**：在 OS 指标上显著超过所有 SOTA 模型（领先 10%-15% 以上），表明结构化语言能极好地帮助模型识别目标。
*   **泛化能力强**：通过解耦底层像素和高层语义，模型在未见过的环境中表现出更强的鲁棒性。

#### 7. 优点
*   **高效性**：极大地降低了模型参数量（缩小 10-50 倍）和推理延迟，适合边缘设备部署。
*   **鲁棒性**：结构化文本过滤了光照、纹理等视觉噪声，使导航决策更关注空间逻辑。
*   **可解释性**：输入是人类可读的文本，便于分析模型在哪个环节（语义识别或空间推理）出现了偏差。

#### 8. 不足与局限
*   **细粒度信息丢失**：网格化处理虽然高效，但会丢失物体的精确形状、纹理和微小障碍物信息，可能影响极复杂场景下的避障。
*   **停止预测缺陷**：虽然 OS 指标极高，但 SR 和 SPL 相对较低，说明模型虽然找到了目标，但在“何时停止”的判断上仍有提升空间。
*   **依赖前端感知**：导航效果高度依赖于预训练语义分割模型的准确性。
