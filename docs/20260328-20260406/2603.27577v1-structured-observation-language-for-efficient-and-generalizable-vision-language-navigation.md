## 摘要
视觉语言导航 (VLN) 要求具身智能体通过遵循自然语言指令在复杂环境中进行导航，这通常需要视觉和语言模态的紧密融合。现有的 VLN 方法通常将原始图像转换为视觉 token 或隐式特征，这需要大规模的视觉预训练，并且在环境变化（如光照、纹理）下泛化能力较差。为了解决这些问题，我们提出了 SOL-Nav（面向导航的结构化观测语言），这是一个将第一人称视角视觉观测转化为紧凑的结构化语言描述的新型框架，旨在实现高效且泛化性强的导航。具体而言，我们将 RGB-D 图像划分为 N*N 网格，提取每个网格单元的代表性语义、颜色和深度信息以形成结构化文本，并将其与语言指令拼接，作为预训练语言模型 (PLM) 的纯语言输入。在标准 VLN 基准测试（R2R、RxR）和真实世界部署中的实验结果表明，SOL-Nav 显著降低了模型规模和对训练数据的依赖，充分利用了 PLM 的推理和表示能力，并在未知环境中实现了强大的泛化性。

## Abstract
Vision-Language Navigation (VLN) requires an embodied agent to navigate complex environments by following natural language instructions, which typically demands tight fusion of visual and language modalities. Existing VLN methods often convert raw images into visual tokens or implicit features, requiring large-scale visual pre-training and suffering from poor generalization under environmental variations (e.g., lighting, texture). To address these issues, we propose SOL-Nav (Structured Observation Language for Navigation), a novel framework that translates egocentric visual observations into compact structured language descriptions for efficient and generalizable navigation. Specifically, we divide RGB-D images into a N*N grid, extract representative semantic, color, and depth information for each grid cell to form structured text, and concatenate this with the language instruction as pure language input to a pre-trained language model (PLM). Experimental results on standard VLN benchmarks (R2R, RxR) and real-world deployments demonstrate that SOL-Nav significantly reduces the model size and training data dependency, fully leverages the reasoning and representation capabilities of PLMs, and achieves strong generalization to unseen environments.

### 1. 核心问题与整体含义（研究动机和背景）
视觉语言导航（VLN）要求智能体根据自然语言指令在复杂环境中移动。**核心挑战**在于如何有效融合视觉和语言模态。
*   **现有问题**：主流方法将图像转换为视觉 Token 或隐式特征，这导致：1) 依赖大规模视觉预训练，训练成本高；2) 对环境变化（光照、纹理、布局）敏感，泛化性差；3) 视觉与语言之间存在模态鸿沟，难以充分利用预训练语言模型（PLM）的推理能力。
*   **研究动机**：探索一种直接将视觉观测转化为“结构化语言”的方法，使导航任务变成纯粹的文本推理任务，从而降低训练门槛并提升跨场景泛化能力。

### 2. 论文提出的方法论：SOL-Nav
SOL-Nav 的核心思想是**将第一人称 RGB-D 图像解构为网格化的文本描述**，并输入轻量级语言模型进行决策。
*   **结构化观测语言（SOL）构建**：
    *   **网格划分**：将图像划分为 $N \times N$ 网格。当前帧使用 $6 \times 6$ 高分辨率，短期记忆使用 $4 \times 4$，长期记忆使用 $2 \times 2$。
    *   **特征提取**：每个网格单元提取三类信息：
        1.  **深度（Depth）**：计算网格内平均深度值。
        2.  **语义（Semantic）**：利用预训练模型（如 SegFormer）识别网格内面积最大的语义类别。
        3.  **颜色（Color）**：将 HSV 空间值映射为标准颜色名称（如“浅灰色”、“蓝色”）。
    *   **文本拼接**：每个单元格表示为 `[i,j]: depth=d, semantic=s, color=c`，所有单元格按顺序拼接成字符串。
*   **模型架构**：
    *   **输入**：系统描述（定义动作空间）+ 结构化观测序列（历史+当前）+ 导航指令。
    *   **骨干网络**：采用 **Qwen3-Embedding-0.6B** 作为编码器。
    *   **输出**：通过多步分类头预测**动作块（Action Chunk）**，一次性输出 4 个连续动作（如：前进、左转、停止等）。

*   **消融实验**：针对“网格分辨率”、“历史信息”和“深度特征”做了三组对比实验，验证了各组件的必要性。
*   **真实世界部署**：在 Unitree Go2 机器人上进行了三个不同场景（茶歇区、大厅楼梯、会议室）的实地测试。
*   **评价**：实验设计较为充分，对比了从轻量级到 7B 以上的大规模模型，且包含了仿真与实测，客观性较强。

### 6. 主要结论与发现
*   **小模型胜大模型**：仅 0.6B 参数的 SOL-Nav 在 R2R-CE 上的表现优于许多 7B 参数的模型（如 UniNaVid），证明了结构化文本表示的高效性。
*   **极强的探索潜力**：SOL-Nav 在 **OS（神谕成功率）** 指标上大幅领先（超过 SOTA 10%-15%），说明模型非常擅长找到目标附近，只是在“停止”动作的判断上仍有提升空间。
*   **泛化能力强**：由于不直接处理像素，模型对光照和纹理变化不敏感，在未知环境中的表现非常稳健。

### 7. 优点（亮点）
*   **模态对齐新范式**：通过将视觉“翻译”成语言，彻底消除了视觉与语言之间的模态鸿沟，无需从头训练视觉编码器。
*   **极低推理成本**：模型规模极小（<1B），可在 Jetson Orin 等边缘设备上实现 0.8s 的低延迟推理，适合具身智能体部署。
*   **高透明度**：输入给 LLM 的是可读的文本，便于开发者理解智能体“看到了什么”以及“为何做出决策”。

### 8. 不足与局限
*   **精细信息丢失**：网格化处理虽然高效，但会丢失物体的形状、纹理等细粒度视觉特征，可能影响在极度复杂场景下的精确导航。
*   **依赖外部感知模型**：SOL-Nav 的性能高度依赖于前端语义分割模型的准确性。
*   **停止判断问题**：虽然 OS 指标很高，但 SR（最终停止成功率）相对较低，说明模型在判断“何时到达终点并停止”这一逻辑上还需优化。
